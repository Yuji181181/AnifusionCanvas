# **キャラクターLoRAファインチューニング詳細要件定義書**

## **1\. 開発の背景とタスクの目的**

アニメ制作のDX化を推進する画像生成AIアプリケーション（中割り生成・編集エディタ）において、「キャラクターのアイデンティティ（デザインの一貫性）」を保つことは最も重要な要件である。

本タスクの目的は、アプリケーション開発とは独立したプロセスとして、特定のオリジナルキャラクターの画風や特徴をAIに学習させるための「LoRA (Low-Rank Adaptation)」学習パイプラインを構築・実行することである。担当するAIエージェントは、限られたリソース下で確実に動作するPythonスクリプトを作成し、開発者が学習を完遂できるようにガイドしなければならない。

## **2\. 前提条件とハードウェア制約**

* **ベースモデル:** Stable Diffusion v1.5ベースのアニメ特化モデル（例: runwayml/stable-diffusion-v1-5 や任意のアニメ系Checkpoint）  
* **ハードウェア制約:** **NVIDIA RTX 3060 (VRAM 8GB)**。  
  * *重要:* VRAM 8GBという厳しい制約下でOOM（Out of Memory）を起こさずに学習を完遂させることが、本タスクの最優先事項である。  
* **フレームワーク:** Hugging Face diffusers ライブラリ、および peft、accelerate をベースとしたPythonスクリプトを一から構築すること。

## **3\. パッケージ管理と環境構築要件**

依存関係の管理には、モダンで高速なパッケージマネージャーである **uv** を使用する。

* 実装担当AIエージェントは、requirements.txt ではなく pyproject.toml （または uv の初期化コマンド群）を出力すること。  
* パッケージを追加・インストールするスクリプトを記述する際は、pip や uv pip install ではなく、必ず **uv add** または **uv run** コマンドを使用する前提で手順を構築すること。

## **4\. データセット自動前処理スクリプト（prepare\_dataset.py）の要件**

学習画像を効率的に処理するため、以下の機能を持つ前処理スクリプトを作成すること。

1. **画像の正規化:** 指定したディレクトリ内の画像を読み込み、SD v1.5の学習に最適な解像度（512x512）に自動的にリサイズ・クロップする処理。  
2. **自動タグ付け（キャプション生成）:** Hugging Faceの SmilingWolf/wd-v1-4-vit-tagger-v2 等（WD14 Tagger）を用いて、画像からアニメ特有のタグ（プロンプト）を自動抽出する。抽出したタグは、元の画像と同名の .txt ファイルとして保存する処理を実装すること。

## **5\. LoRA学習スクリプト（train\_lora.py）とVRAM最適化要件**

VRAM 8GB制約をクリアするため、以下の最適化技術を**必ずすべて**コード内に組み込むこと。

1. **Mixed Precision:** mixed\_precision="fp16" （半精度浮動小数点での計算）を有効化する。  
2. **Gradient Checkpointing:** gradient\_checkpointing=True を有効化し、計算グラフのメモリ消費を節約する。  
3. **8-bit Optimizer:** オプティマイザには bitsandbytes ライブラリの AdamW8bit を使用し、オプティマイザの状態（State）によるメモリ消費を大幅に削減する。  
4. **Memory Efficient Attention:** VRAM消費を抑えつつ高速化するため、xformers によるAttention最適化を有効化する。  
5. **Batch SizeとGradient Accumulation:** VRAM制約をクリアするため、実バッチサイズは train\_batch\_size=1 と設定する。不足する実質的なバッチサイズを補うため、gradient\_accumulation\_steps=2 または 4 を設定して勾配を蓄積するアーキテクチャとすること。

## **6\. ハイパーパラメータの指定**

スクリプトには、以下の設定値をデフォルトの引数または変数として組み込むこと。

* **U-Net Learning Rate:** 1e-4 (0.0001)  
* **Text Encoder Learning Rate:** 5e-5 (0.00005)  
* **LoRA Rank (dim) / Alpha:** dim=32, alpha=16  
* **Total Training Steps:** 約1500ステップ（過学習と学習不足を防ぐスイートスポット）を目標値とする。  
* **チェックポイント保存:** 500ステップごとに学習の途中経過を .safetensors 形式で保存するロジックを実装すること。

## **7\. 納品物要件とエラーハンドリング**

実装担当AIエージェントは、最終的な成果物として以下の情報を提供すること。

1. **環境構築手順:** uv init や uv add を使用した完全なセットアップコマンド群。  
2. **完全なコード:** prepare\_dataset.py および train\_lora.py の実行可能なPythonコード。  
3. **トラブルシューティング:** VRAM不足（OOM）エラーや、xformers のビルドエラー等が発生した場合の、具体的な回避策・トラブルシューティングガイド。
