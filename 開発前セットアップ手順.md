# 開発前セットアップ手順（ファインチューニングなし）

この手順は、開発着手前に必要な外部連携をまとめたものです。

## 0. ローカル前提ツール

### 必須
- bun
- uv
- Python 3.11+
- git

### 外部連携用
- modal CLI
- cloudflared CLI
- aws CLI（R2をS3互換で検証するため）
- ffmpeg（推論後の動画化）

確認コマンド:

```bash
uv --version
bun --version
modal --version
cloudflared --version
aws --version
ffmpeg -version
```

## 1. バックエンド環境変数

```bash
cd backend
cp .env.example .env
```

`.env` に最低限設定する項目:
- `DATABASE_URL`
- `R2_ACCOUNT_ID`
- `R2_ACCESS_KEY_ID`
- `R2_SECRET_ACCESS_KEY`
- `R2_BUCKET`
- `R2_ENDPOINT_URL`
- `HF_TOKEN`（private/gated model を使う場合）
- `TOONCRAFTER_MODEL_ID`
- `SD_INPAINT_MODEL_ID`

## 2. TiDB Serverless

### 手順
1. TiDB Cloudで Serverless クラスターを作成
2. Database / User を作成
3. 接続文字列を取得し `DATABASE_URL` に設定

例:

```env
DATABASE_URL=mysql+pymysql://USER:PASSWORD@HOST:4000/DB_NAME
```

### よくあるNG（今回の症状）
- `mysql://...` になっている
	- この形式は `MySQLdb` ドライバを使おうとするため、`No module named 'MySQLdb'` になります。
	- `mysql+pymysql://...` に修正してください。
- パスワードに `@` や `:` が入っている
	- URLエンコードが必要です（例: `@` は `%40`）。
- DB名が未作成
	- TiDB Cloud 上で DB を先に作成してください。
- TLS必須設定を使う環境
	- 必要なら `?ssl_verify_cert=true` を末尾に付けてください。

修正版の例:

```env
DATABASE_URL=mysql+pymysql://USER:PASSWORD@HOST:4000/DB_NAME?ssl_verify_cert=true
```

### 接続確認

```bash
cd backend
uv run python scripts/preflight_check.py
```

`[OK] tidb` が出れば接続成功です。

## 3. Cloudflare R2

### 手順
1. Cloudflare Dashboard にログイン
2. 左メニューから `R2` を開く
3. `Create bucket` を押してバケットを作成
	- 例: `anifusion-canvas-dev`
	- 作成直後に表示されるバケット名を `.env` の `R2_BUCKET` に入れる
4. 同じく `R2` 画面で `Manage R2 API tokens` を開く
5. `Create API token` を押す
	- テンプレートは `Object Read & Write` を選択
	- 対象バケットは作成したバケットを指定
6. 作成完了時に表示される以下2つを控える（この画面を閉じると再表示不可）
	- `Access Key ID` → `.env` の `R2_ACCESS_KEY_ID`
	- `Secret Access Key` → `.env` の `R2_SECRET_ACCESS_KEY`
7. `R2` トップ画面または URL から `Account ID` を確認
	- `.env` の `R2_ACCOUNT_ID` に設定
8. `R2_ENDPOINT_URL` を以下形式で設定
	- `https://<R2_ACCOUNT_ID>.r2.cloudflarestorage.com`
9. `.env` に以下を設定

```env
R2_ACCOUNT_ID=...
R2_ACCESS_KEY_ID=...
R2_SECRET_ACCESS_KEY=...
R2_BUCKET=...
R2_ENDPOINT_URL=https://<R2_ACCOUNT_ID>.r2.cloudflarestorage.com
R2_REGION=auto
```

### 「R2のコードはどこから？」への回答
- このプロジェクトで必要なのは、Cloudflareの「コード」ではなく認証情報です。
- 取得元は Cloudflare Dashboard の `R2` 画面です。
	- `R2_ACCESS_KEY_ID` / `R2_SECRET_ACCESS_KEY`: `Manage R2 API tokens` で作成時に表示
	- `R2_ACCOUNT_ID`: ダッシュボードのアカウント情報
	- `R2_BUCKET`: 自分で作成したバケット名
	- `R2_ENDPOINT_URL`: 上記 `R2_ACCOUNT_ID` から自分で組み立て

### CORS 推奨設定（ブラウザアップロード想定）
- Origin: `http://localhost:3000`
- Methods: `GET,PUT,POST,HEAD`
- Headers: `*`

設定場所:
1. `R2` > 対象バケット > `Settings`
2. `CORS policy` の編集を開く
3. 上記ルールを保存

### 接続確認

```bash
cd backend
uv run python scripts/preflight_check.py
```

`[OK] r2` が出れば接続成功です。

## 4. OSSモデルの取得（ファインチューニングなし）

このデモではモデルは推論専用で使用します。

### 手順
1. Hugging Face にログイン（未登録なら新規作成）
2. 右上プロフィールアイコン > `Settings` > `Access Tokens`
3. `New token` を作成
	- Name: 任意（例: `anifusion-local-dev`）
	- Role: `Read`
4. 発行されたトークン（`hf_` で始まる文字列）をコピー
5. `.env` の `HF_TOKEN` に設定
6. 使用モデルページを開き、gated の場合は `Agree` / `Request access` を実施
7. `.env` にモデルIDを設定
8. ダウンロード実行

```bash
cd backend
uv run python scripts/download_models.py
```

### 高速ダウンロード（推奨）
このリポジトリでは `aria2c` を使ってモデルを高速取得します。

```bash
cd backend
export ARIA2_SPLIT=4
export ARIA2_MAX_CONNECTION_PER_SERVER=4
export ARIA2_MAX_CONCURRENT_DOWNLOADS=1
export ARIA2_RETRY_WAIT=5
export ARIA2_MAX_TRIES=20
export ARIA2_TIMEOUT=30
uv run python scripts/download_models.py
```

回線が安定している場合だけ `ARIA2_SPLIT` を `8` 以上に上げてください。
Wi-Fiが落ちる場合は `ARIA2_MAX_CONNECTION_PER_SERVER=2` まで下げると安定しやすいです。

### 補足
- ダウンロード先は `MODEL_CACHE_DIR`（デフォルト: `./artifacts/models`）
- gated/private model は `HF_TOKEN` が必須

### HF_TOKEN の確認
```bash
cd backend
uv run python -c "from huggingface_hub import HfApi; import os; print('HF_TOKEN set:', bool(os.getenv('HF_TOKEN'))); print(HfApi().whoami(token=os.getenv('HF_TOKEN'))['name'])"
```
